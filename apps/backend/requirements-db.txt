## Setup Instructions

### 1. **Initialize Alembic** (if not already done)

```bash
cd apps/backend
alembic init migrations
```

### 2. **Configure alembic.ini**

Update `alembic.ini`:

```ini
# alembic.ini configuration
sqlalchemy.url = driver://user:pass@localhost/dbname
script_location = migrations
sqlalchemy.echo = false
sqlalchemy.echo_pool = false
sqlalchemy.pool_pre_ping = true
sqlalchemy.pool_recycle = 3600
```

### 3. **Configure env.py**

The `migrations/env.py` file is already configured to:
- Load environment variables from `.env`
- Support both online and offline migrations
- Use SQLAlchemy ORM models

### 4. **Create Initial Migration**

```bash
alembic revision --autogenerate -m "Initial schema with users, auth, and audit tables"
```

### 5. **Run Migrations**

```bash
# Upgrade to latest version
alembic upgrade head

# Upgrade to specific version
alembic upgrade 003

# Check current migration status
alembic current

# Show migration history
alembic history --verbose
```

### 6. **Rollback Migrations**

```bash
# Downgrade one migration
alembic downgrade -1

# Downgrade to specific version
alembic downgrade 001

# Downgrade all (be careful!)
alembic downgrade base
```

## Docker Deployment

### Run migrations in Docker

```bash
# One-time migration run
docker-compose exec backend alembic upgrade head

# Check migration status
docker-compose exec backend alembic current

# View migration history
docker-compose exec backend alembic history
```

### Include migrations in Docker startup

Update `docker-compose.yml`:

```yaml
services:
  backend:
    build: ./apps/backend
    environment:
      DATABASE_URL: postgresql://ra_user:ra_password@postgres:5432/ra_db
    depends_on:
      postgres:
        condition: service_healthy
    command: >
      sh -c "alembic upgrade head &&
             uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload"
```

## Migration Best Practices

1. **Always use `--autogenerate`** for model changes
2. **Review generated migrations** before applying
3. **Test migrations locally** before production
4. **Keep migrations small** (one logical change per migration)
5. **Never modify** migrations in production branch
6. **Always provide** descriptive migration messages
7. **Test rollbacks** to ensure downgrade works
8. **Document breaking changes** in migration comments

## Troubleshooting

### Migration conflicts

```bash
# Merge conflicting migrations
alembic merge -m "Merge heads after conflict"

# View migration branches
alembic branches
```

### Database already at head

```bash
# Show current version
alembic current

# Show available migrations
alembic history --verbose

# Downgrade and re-apply to test
alembic downgrade -1
alembic upgrade +1
```

### Reset everything (development only)

```bash
# Drop all tables
psql -d ra_db -c "DROP SCHEMA public CASCADE; CREATE SCHEMA public;"

# Recreate from scratch
alembic upgrade head
```
